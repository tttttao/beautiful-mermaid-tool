## 2024-05-19 - Save Button Visual Feedback and Accessibility
**Learning:** Adding temporary visual feedback (swapping the save icon for a checkmark) greatly improves user confidence in actions. However, dynamically changing visual icons without announcing them to screen readers creates an inaccessible experience. Setting `aria-live="polite"` on the parent container alongside dynamically bound `:aria-label` and `:title` ensures the temporary state change ("Chart saved") is read by screen readers seamlessly without moving focus.
**Action:** Always combine temporary visual feedback changes with `aria-live="polite"` and dynamic `aria-label`/`title` attributes to maintain accessible parity for micro-interactions. Ensure any `setTimeout` used for reverting state correctly clears existing timers to prevent state collisions.
