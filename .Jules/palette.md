## 2024-03-29 - Dynamic Accessibility Labels and Color Scoping for Visual Feedback
**Learning:** When providing visual feedback on buttons (e.g., swapping a 'Save' icon for a 'Check' icon), it's crucial to update `title` and `aria-label` dynamically to match the current visual state so screen readers announce it properly. Adding `aria-live="polite"` ensures the state change is announced interactively. Additionally, when using utility-first CSS like Tailwind, applying specific color classes directly to the child icon element (rather than appending conditional classes to the parent button) avoids collisions with the parent's existing `hover`/`focus` state classes.
**Action:** When implementing visual feedback on interactive elements, always bind `title` and `aria-label` to the same state as the visual change, add `aria-live="polite"` to the container, and target CSS color utility classes specifically to the child icon to maintain clean interaction states.
