## 2024-05-03 - Accessible Transient Feedback States
**Learning:** When implementing temporary visual feedback states on buttons (like swapping a save icon to a checkmark), appending a text color class to the parent button can cause Tailwind precedence collisions or break internal element coloring. Furthermore, visually swapping an icon isn't communicated to screen readers if focus doesn't move.
**Action:** Combine dynamic `aria-label`/`title` attributes and `aria-live="polite"` on the button container to ensure screen readers announce the state change inline. Apply color classes directly to the new icon component, and add `aria-hidden="true"` to decorative/redundant icon elements. Always use `clearTimeout` before setting a new timeout to clear the state to prevent rapid clicks from causing buggy overlapping resets.
