## 2025-02-27 - Dynamic Status Icons and Accessibility
**Learning:** When swapping out icons to indicate a temporary state change (like showing a "Check" instead of a "Save" icon to confirm an action), the `title` and `aria-label` attributes on the parent `<button>` must be simultaneously updated. Otherwise, screen readers will incorrectly announce the button's old state, or just the old name, causing confusion. Furthermore, when adding visual feedback colors in Tailwind, applying the color classes directly to the icon elements (`text-emerald-500`) instead of the parent button avoids Tailwind class precedence collisions with hover/focus states.
**Action:** Always bind `title` and `aria-label` to the same state variable controlling dynamic icon swaps, and apply state-specific color classes directly to the SVG/icon component rather than conditionally appending them to the parent button's class list.
