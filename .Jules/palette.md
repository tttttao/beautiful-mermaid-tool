
## 2024-03-27 - Icon Color Overrides and ARIA Syncing
**Learning:** When swapping Lucide icons inside standard buttons to convey state (like "Saved"), applying dynamic color utility classes directly on the `<svg>` icon component (e.g., `<Check class="text-emerald-500" />`) is vastly superior to applying those same classes to the parent `<button>` container. Attempting to override text colors on a parent container with complex state transitions in Tailwind frequently results in CSS specificity conflicts (especially when `focus:`, `hover:`, and `dark:` variants are involved), leading to visual bugs. Additionally, whenever a visual icon is changed to convey status, the `title` and `aria-label` attributes on the parent button must be simultaneously updated so screen readers properly announce the dynamic state transition.
**Action:** Always apply specialized state colors directly to the icon element to ensure they reliably supersede parent styling, and enforce a strict policy of updating `aria-label`/`title` alongside any visual state changes.
